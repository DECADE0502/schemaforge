# SchemaForge 系统级设计目标

## 核心原则

**AI 只做意图理解，所有工程决策通过本地确定性接口完成。**

### AI 负责的事

- 从自然语言提取模块意图（器件型号、电路类别、电压/电流目标）
- 推断模块间连接意图（电源链、GPIO 控制、总线连接）
- 识别歧义和不确定项
- 视觉审稿（评判原理图布局质量，不改电气）

### AI 绝对不做的事

- 不计算电阻/电容/电感数值（本地公式引擎做）
- 不决定原理图坐标（本地布局引擎做）
- 不生成 SVG/BOM/SPICE（本地渲染/导出引擎做）
- 不做引脚级连接映射（本地规则引擎做）
- 不替换用户指定的精确型号

## 数据流

```
用户自然语言
    │
    ▼
[AI 意图解析] → SystemDesignRequest (模块意图 + 连接意图)
    │
    ▼
[本地器件解析] → ModuleInstance[] (器件命中 + 端口映射)
    │
    ▼
[本地连接规则] → ResolvedConnection[] + SystemNet[] (引脚级连接)
    │
    ▼
[本地参数综合] → 每个模块的外围参数 (公式驱动)
    │
    ▼
[本地全局编号] → 全局元件实例表 (C1-Cn, R1-Rn, U1-Un)
    │
    ▼
[本地系统渲染] → 单张 SVG (所有模块 + 连线 + 标签)
    │
    ▼
[本地 BOM/SPICE] → 合并输出
    │
    ▼
[本地规则审查] → 42+ 条工程审查
    │
    ▼
[AI 视觉审稿] → 布局优化建议 (仅限白名单动作)
    │
    ▼
[本地 Patch 执行] → 重渲染 → 循环
```

## 目标场景

用户输入：
> "20V输入，用TPS54202降压到5V，再用AMS1117降到3.3V，给STM32F103C8T6供电，并且用MCU的PA1控制一颗LED"

系统输出：
- 一张完整 SVG：TPS54202 Buck + AMS1117 LDO + STM32 + LED + 限流电阻
- 四个模块之间有电气连线
- 合并 BOM（全局编号）
- 合并 SPICE（共享节点）
- 工程审查报告
